Mode opératoire de l’application SUSAR_EU_V2


Introduction

L’application SUSAR_EU_v2 vise à remplacer un précédent outil développé sous Access (SUSAR_EU) qui ne permettait pas son utilisation en télétravail. De plus, l’importante quantité de données après plusieurs années d’utilisation le rendait très lent. Ces deux problèmes sont dans résolus avec cette nouvelle version.

SUSAR_EU_v2 est une application développée en PHP/Symfony, les utilisateurs y accèdent depuis un navigateur (Firefox et Chrome ont été testés), via l’URL suivante :

http://172.16.63.170/susar_eu_v2/public/index.php/

L’utilisateur est alors invité à se connecter, le login est son adresse mail ANSM.

Si l’utilisateur n’a pas encore de compte actif, il peut en faire la demande directement auprès de l’administrateur de l’application : Frederic.RANNOU@ansm.sante.fr

II – Import des fichiers CTLL - Action DMFR



Cette étape est nécessaire car elle permet de récupérer, via EUDRA vigilance, la liste des substances « synonymes » aux « high-level substance name » renseignées par les pilotes VEC lors de l’étape précédemment décrite.

Une fois ces données récupérées puis chargées dans SUSAR_EU_v2, elles seront utilisées par le script d’import automatique qui tourne toutes les nuits. Ce script permet d’alimenter la base SUSAR_EU_v2 à partir des données issues de la BNPV en requêtant par « low-level ».

Ainsi on récupère tous les cas y compris si la substance change de nom (passage d’un code produit a une DCI, formatage différent selon les pays, …).

III – Évaluation des SUSARs - Action DMFR



Après importations des SUSARs, par la DMFR, des fichiers CTLL provenant de l’EMA, les cas France/saMS et Mono-nationaux sont disponibles pour évaluation dans SUSAR_EU_v2.



A – Écran « liste des SUSARs » - présentation de l’écran



Cet écran est accessible via le menu « Liste des SUSARs » :

  1. Champs de recherche

Les différents champs de recherche ont été regroupés par thèmes afin d’améliorer la lisibilité.

Liste des champs de recherche :

  • Cas archivés (traités dans SUSAR_EU_v1) : par défaut, est sélectionné « Uniquement les cas non-archivés ». Cela correspond aux cas importés dans la base après la migration susar_eu => susar_eu_v2 (vendredi 23/05/2025 au soir), ainsi que les cas d’origine Access à partir du 16/04/2025. Avec ce menu déroulant, on peut donc visualiser :

    • Uniquement les cas non-archivés

    • Uniquement les cas archivés

    • Tous les cas

  • Évaluateur référent : Évaluateur DMM ou DEI attribué aux SUSARs lors de l’import

  • DMM/Pôle : Direction et pôle attribué aux SUSARs lors de l’import

  • Cas traité : SUSAR ayant été évalué ou non

  • Évalué par : Évaluateur ayant évalué le cas

  • Conclusion évaluation : Évaluation renseignée (Concern in CT, Monitor, …)

  • N° EUCT : Numéro EUDRA CT de l’essai clinique

  • Sponsor study num. : Référence du promoteur de l’essai clinique

  • Pays survenue : Pays ou le SUSAR a été constaté

  • Niveau de classification : Niveau de classification du cas attribué lors de l’import (Niveau 2a : décès en Europe ou pronostic vital engagé en Europe, …)

  • Substance active : Subtance active suspecte, en anglais, du susar

  • PT : Effet indésirable, selon le niveau PT meddra en anglais du susar

  • idSUSAR : identifiant unique d’un cas dans la base de données de SUSAR_EU_v2

  • ID cas (worldwide) : worldwide id du cas, cette information permet nottamment de rechercher le cas en question dans les différents systèmes d’information.

  • Date décla. (début et fin) : fourchette de dates correspondant a la gateway date, c’est-à-dire la date de déclaration du cas dans EUDRA

  • saMS/Mono : cas dont la France est saMS sur la substance ou bien il s’agit d’une substance mono-nationale.

  • Population : Type de population (Paediatric, Adult, …)

  • Cas Europe : Cas survenu en Europe

  • Cas DME : Effet indésirable présent dans la liste des DME

  • Cas IME : Effet indésirable présent dans la liste des IME

  • Narratif : Recherche textuelle d’un mot dans les 3000 premiers caractères du narratif

Il existe de plus 3 boutons permettant de réaliser des actions en rapport aux différents champs de recherche :

  • Rechercher : Lance la recherche selon les critères renseignés

  • Export Excel : Après qu’une recherche ait été lancée, ce bon permet d’exporter la liste des SUSARs obtenue sous un format Excel. Le fichier est automatiquement téléchargé lorsque la requête est terminée.

  • Reset : Permet de remettre les champs de recherche a l’état initial : tous les champs a vide, sauf Cas archivés (traités dans SUSAR_EU_v1) à « Uniquement les cas non-archivés ».











  1. Liste des SUSARs

Les différents susars s’affichent dans des rectangles alternativement mauves et violets, pour les susar actifs, d’origine SUSAR_EU_v2 et rouge clair et rose pour les susar inactifs, d’origine SUSAR_EU_v1 (base Access).

Les informations affichées dans cette liste de cas correspondent aux champs de recherche décrits plus haut, avec néanmoins quelques ajouts :

  • Serioussness : Critère(s) de gravité du cas

  • Tableau « suspect/interacting » : Informations détaillées sur les médicaments suspects ou en interactions présentes dans les fichiers CTLL.

  • Tableau reaction PT : Information détaillées sur les effets indésirables présents dans les fichiers CTLL.

  • Bouton « Afficher les antécédents médicaux » : Permet d’afficher un tableau des « medical history » présents dans les fichiers CTLL. Il a été choisi de masquer ces données par défaut.

  • Lien ICSR : il s’agit d’un lien hypertext vers la fiche ICSR présente sur le site de l’EMA. Pour s’y connecter l’utilisateur doit être titulaire d’un compte EMA.

Il existe de plus 3 boutons permettant de réaliser des actions liées a l’évaluation du cas. Leurs effets seront décrits dans le paragraphe suivant.

  • Autres F.U. : Permet d’ouvrir un nouvel onglet dans le navigateur pour afficher tous les « case version » d’un même cas présent dans la base.

  • Assessed without action : permet d’évaluer en un clique, tous les couples substance/PT d’un cas a « assessed without action »

  • Détail/Éval. : Permet d’ouvrir un nouvel onglet dans le navigateur et d’y afficher le détail d’un SUSAR.

  1. Bouton Détail/Éval.

L’appui sur ce bouton déclenche l’ouverture d’un nouvel onglet avec les informations détaillées du SUSAR.

Les évaluations sont affichées dans le paragraphe orange. Pour un SUSAR non évalué, l’affichage est le suivant :

Il est alors possible d’ajouter une évaluation grâce au bouton bleu « Ajout d’une évaluation », cela ouvre alors l’écran suivant :

Il convient alors de renseigner un « assessment outcome » pour le couple substance / PT sélectionné grace aux deux menus déroulants du haut.

Il est également possible de renseigner un commentaire.

La validation va déclencher la création des évaluation a « Assessed without action » pour les autres couples SA/PT du SUSAR et avertir l’utilisation grâce à l’affichage du message suivant :

Le paragraphe orange est alors complété :

Il est alors possible de modifier ou de supprimer chacune des évaluations d’un couple donnée.

Pour quitter cet onglet du détail d’un cas, il est conseillé de cliquer sur le bouton « Fermer cet onglet ». Cela permettra de déclencher la mise à jour de l’onglet présentant la liste des SUSARs.



Si le cas est attribué à plusieurs évaluateurs, l’appuie sur le bouton « Détail/Éval. », la pop-up suivante, s’affiche :

Le nom du ou des autres évaluateurs est mentionné afin de pouvoir les contacter et ainsi rendre un avis collégial.

  1. Bouton Autres F.U.



Ce bouton permet d’ouvrir un nouvel onglet affichant les différentes « case version » d’un même « WorldWide id » présentes dans la base de données de SUSAR_EU_v2.

Il est ainsi possible de visualiser le travail précédemment fait pour ce cas.

Les différents follow-up sont triés avec le plus récent en haut.

Les évaluations sont alors possibles selon les deux méthodes décrites précédemment :

  • Bouton Assessed without action

  • Bouton Détail/Éval.



Mais également par une troisième méthode « Évaluation en masse des Follow-up », en effet, pour les susars affectés à un seul évaluateur, un petit formulaire spécifique est visible dans la partie haute de cet écran :

L’utilisateur peut sélectionner une substance et un PT puis une conclusion et enfin cliquer sur le bouton « Appliquer l’évaluation ». Cela va, pour chaque susar ayant le World Wide ID courant (=les différents follow up), qui ne sont pas déjà évalués, affecter l’Assessment outcome sélectionné au couple SA/PT sélectionné. Les autres couples seront évalués à « Assessed without action ».

Si l’utilisateur sélectionne « toutes substances » ou « tout PT », l’évaluation sélectionnée sera appliquée à ces couple SA/PT.

Comme il est précisé plus haut, les SUSARs déjà évalués ne sont pas affectés par cette évaluation en masse.
